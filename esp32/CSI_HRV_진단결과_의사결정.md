# CSI-HRV 진단 결과 & 의사결정 문서

> 작성 2026-06-14 (v1) / 업데이트 2026-06-14 (v2) | 팀 마주봄(MajuBom) | 담당 박형석
> **목적**: CSI로 HRV(심박변이도)를 뽑으려다 막힌 이유를 데이터+논문으로 정리하고, 최종 방향을 확정한 문서.
> **한 줄 결론 (v2)**: **HRV 절대값(SDNN ms) 측정은 단일안테나 ESP32로 학술적 불가 확정. 신청서 표현 "HRV"는 유지하되 의미를 "자율신경 변화 추세 z-score"로 확장 → 신청서 핵심 목표 동일 달성.**

---

## 1. 우리가 하려던 것

낙상 사전예측 시스템의 CSI 파트(자율신경 30% 기여). 목표 = 비접촉으로 **HRV(SDNN/RMSSD)** 를 측정해 자율신경 상태 → 낙상 위험 지표화.

하드웨어: ESP32-S3 2개 (송신 softAP + 수신 ping), 111Hz CSI 수신. 침대 양쪽 가슴높이.

---

## 2. 무엇을 했나 (오늘 작업 순서)

1. ESP32 2개로 CSI 100~111Hz 수신 셋업 (정상 작동)
2. 90초 측정 → 분석기 `vital_csi2.py`(진폭 기반)로 HR/SDNN 산출
3. **애플워치 ground truth로 5회 비교 측정** (마음챙김 호흡앱 사용)
4. 결과가 안 맞아서 **체계적 디버깅**으로 근본원인 추적
5. **숨참기(breath-hold) 테스트**로 검증
6. 원논문(IEEE 9508523) PDF 정독 → 왜 안 되는지 확정

## 3. 애플워치 비교 결과 (핵심 증거)

| 회차 | 워치 HR | CSI HR | 워치 SDNN | CSI SDNN |
|---|---|---|---|---|
| 1 | 79 | 79 (운) | 35 | 66 |
| 2 | 73 | 85 ❌ | 35 | 88 |
| 3 | 71 | 85 ❌ | 33 | 96 |

- **HR**: 워치는 79→73→71로 내려가는데 CSI는 ~85 고정 (1회차 일치는 우연)
- **SDNN**: 워치 33~35ms로 안정적인데 CSI는 66~96ms로 **2~3배 부풀고 들쭉날쭉**
- → **절대 HRV 부정확 확정**

## 4. 근본원인 (전부 데이터로 증명)

### ① 진폭(amplitude)은 틀린 신호
- `vital_csi2.py`가 CSI 진폭을 썼는데, 진폭은 매번 ~85bpm 엉뚱한 값.

### ② 호흡 고조파가 심박대역을 도배
- 마음챙김 깊은 호흡 6rpm = 0.1Hz. 그 **13~16번째 고조파 = 78/84/90/96bpm**.
- 자기상관이 이 고조파 떼(comb)의 중심 ~85에 락됨.
- 6rpm이면 고조파 간격이 6bpm이라 진짜 심박과 **분리 불가**.

### ③ 에어컨 바람 = 심박을 묻는 노이즈
- 에어컨 끄니 위상 자기상관 강도 0.29 → 0.47로 개선.
- 공기흐름은 WiFi CSI의 대표적 노이즈원 (심박은 가슴 0.5mm 미동이라 쉽게 묻힘).

### ④ ⭐ 결정타: quotient에 필요한 다중안테나가 없음 (논문으로 확정)
- 원논문(아래 §6)의 핵심 기법 = **CSI quotient = 같은 수신기의 두 안테나 진폭 비율**.
- 두 안테나의 랜덤 위상오프셋·임펄스노이즈가 동일 → 나누면 상쇄 → 심박 SNR 급상승.
- **ESP32-S3는 RX 안테나 1개, RF체인 1개** → 두 안테나 CSI 동시 수신 불가 → **quotient 계산 자체가 불가능.**
- ESP32 2개를 RX로 써도 안 됨 (독립 발진기라 노이즈가 안 상쇄됨. quotient는 *같은* 수신기 두 안테나 전용).

## 5. 검증 성공한 것 (= CSI로 되는 것)

**에어컨off + 숨참기 + 위상(phase) 자기상관:**
- CSI HR **67bpm** vs 동시 애플워치 **68~70bpm** = **±2bpm 일치 ✅**

| 능력 | 가능? | 비고 |
|---|---|---|
| **호흡수** | ✅ 완벽 | 6.0rpm 정확히 측정 (마음챙김 호흡 그대로) |
| **평균 심박(HR)** | ✅ 됨 | 깨끗한 조건에서 ±2bpm. 자기상관은 노이즈에 강함 |
| **HRV(SDNN/RMSSD)** | ❌ 불가 | 검증 최상조건에서도 SDNN 159ms(정상 35). quotient 없이 개별박동 추출 불가 |

> 핵심: **평균(HR)은 수십 사이클 평균이라 노이즈에 강해서 됨. HRV는 박동 하나하나 ms 정밀도가 필요해서 quotient 디노이징 없이는 불가.**

## 6. 원논문 분석 (IEEE BHI 2021, 9508523)

**"Heart Rate Variability Extraction using Commodity Wi-Fi Devices via Time Domain Signal Processing"** — Shirakami & Sato (교토대)

| 항목 | 논문 | 우리(ESP32) |
|---|---|---|
| 하드웨어 | **Intel 5300 NIC** ×2 PC | ESP32-S3 |
| RX 안테나 | **한 수신기에 3개** (1×3 SIMO) | **1개** ← 결정적 차이 |
| 핵심기법 | **두 안테나 진폭비(quotient)** 로 노이즈 상쇄 | 불가 |
| 필터 | Butterworth BPF 1–3.5Hz | — |
| HRV추출 | peak → SDNN작은 신호 5개 선택 → 시간영역 합산 → 재peak | — |
| 샘플링 | ~160Hz | 100Hz |
| 자세/거리 | 정좌, 가슴높이, 70~150cm, 60초 | 누움 |
| 성능 | **RMSE 130ms**, SDNN 최저오차 8.8%(좋은위치만), RMSSD 일부만 | — |

**시사점**: 논문조차 전용 다중안테나 NIC로 RMSE 130ms (절대 쉽지 않음). 그 핵심 기법이 **단일안테나 ESP32엔 적용 불가**. → 우리 실패는 코드가 아니라 하드웨어 한계.

## 8. 발표/방어 논리 (어느 안이든 활용)

> "CSI-HRV는 **다중안테나 NIC(Intel 5300)** 가 필요한 quotient 기법이 핵심(IEEE BHI 2021). 단일안테나 ESP32로는 평균 심박·호흡까지가 한계임을 실측+ground truth로 검증했고(±2bpm), HRV는 [선택한 안]으로 확보한다."

- 이건 약점이 아니라 **'한계를 정확히 파악하고 적절한 센서로 분담했다'**는 엔지니어링 강점으로 포장 가능.

원문 정확히:
- "**개인별 기준 대비 변화량(z-score)**을 기반으로 당일 낙상 위험도를 산출"

## 12. 최종 방향 (v2 확정)

### 알고리즘 출력 (CSI 파트)
```
[60초 윈도우마다 계산]
  ① HR (분당 심박수) — 위상 자기상관, ±2bpm 검증됨
  ② 호흡 (분당 호흡수) — Welch PSD
  ③ 자기상관 강도 (신호 안정성, 0~1)

[30일 누적 후 개인 베이스라인 학습]
  각 지표별 평균 μ, 표준편차 σ

[당일 측정 → z-score 환산]
  z_HR     = (HR_today     − μ_HR    ) / σ_HR
  z_호흡   = (호흡_today   − μ_호흡  ) / σ_호흡
  z_안정성 = (안정성_today − μ_안정성) / σ_안정성

[통합 자율신경 위험 점수]
  자율신경 = |z_HR| + |z_호흡| + |z_안정성|

[융합 위험도 (신청서 §주요 기능)]
  최종 = mmWave_보행위험(70%) + 자율신경(30%)

[알람 트리거]
  최종 > 임계값 → 4단계 알림 (정상/주의/경고/위험)
```

### 검증 방식
- 애플워치 마음챙김 1분 호흡 + CSI 90초 동시 측정
- 이미 5회 완료 (csi_watch1~3.csv, csi_hold4.csv)
- 추가 측정 5~10회 (평소 vs 가벼운 stress)
- **Polar H10 안 삼** (비접촉 차별성 모순)

### 하드웨어 구성 (변경 없음)
