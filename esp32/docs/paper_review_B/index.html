🔧 실험 셋업 — 우리 환경과 거의 동일

하드웨어 (우리랑 거의 같음 ⭐)

  • • ESP32-WROOM-32 × 2개 (Xtensa dual 240MHz, 520KB SRAM)
  • • TX: beacon frame ~10 frames/sec 송신
  • • RX: monitor 모드 sniff + CSI 추출 (esp-idf 커스텀 펌웨어)
  • • CSI 샘플링 100Hz
  • • 30 subcarrier 사용
  • • 동기화: Linux NTP daemon (오프셋 <50ms)
⚠️ Reference Device: Fitbit Charge 5 (좌측 손목), 클라우드 API로 HRV 메트릭 자동 수집

실험 프로토콜 (총 14분)

① Baseline 5분

의자에 앉아 정지, 자연호흡 (12~16/min)

② Stress 4분

100에서 7씩 빼기 (Serial Subtraction)
→ 교감신경 항진, LF/HF 상승 예상

③ Recovery 5분

메트로놈 가이드, 5회/분 호흡
→ 부교감 회복

참가자: 14명 (남 9, 여 5, 22~46세)

거리: TX-RX 3m, 사무실 20~22°C

⚙️ 신호처리 파이프라인 — 7단계 (논문 본문 §3 그대로)

1

CSI 획득 + 위상 차분 (식 1, 4)

2.4GHz CSI 100Hz 수집. 위상은 슬로우 페이딩 제거를 위해 차분:

Δφ_diff(t) = ∠H_k(t) − ∠H_k(t−1) − θ_offset

θ_offset은 baseline 첫 5초 정지 구간에서 추정.

2

Detrending — 하드웨어 드리프트 제거 (식 5)

선형 회귀로 저주파 드리프트 빼기:

y_detrended(t) = y(t) − (a·t + b)
3

BPF — 0.8–2.5Hz, 4차 Butterworth IIR (식 6)

심박 대역 통과. HR 48~150BPM 범위.

⚠️ 논문 A와 다름! 논문 A는 1–3.5Hz, 2차. 논문 B는 0.8–2.5Hz, 4차. → 우리 BPF 설정 시 어느 쪽을 따를지 결정 필요. (호흡 잔재 영향 검토)
4

적응형 Peak 검출 (식 7) ⭐ 핵심 기여

시간 변동 임계 + 기울기 영점:

Peak at t_i if y_BP(t_i) > μ + k·σ AND dy_BP/dt|_{t_i} ≈ 0
  • • μ, σ = 30초 슬라이딩 윈도우의 로컬 평균·표준편차
  • • k = 1.5 (empirically tuned, 14명 실험에서 안정적)
  • • RRI ≥ 400ms 제약 (= HR ≤ 150BPM) → 거짓 peak 제거
5

IBI 추출 + IQR Outlier 제거 (식 8, 9) ⭐ 우리 vital_csi2.py에 없는 단계

IBI_i = t_{i+1} − t_i
IBI_i 제거 시점: |IBI_i − median(IBI)| > 1.5 × IQR(IBI)

평균적으로 한 세션당 ~5% 제거. 이게 RMSSD 부풀림 직접 잡는 후처리.

6

시간영역 HRV (식 10~12)

SDNN = √(1/(N−1) · Σ(IBI_i − IBI̅)²)
RMSSD = √(1/(N−1) · Σ(IBI_{i+1} − IBI_i)²)
CV = SDNN / IBI̅ × 100%
7

주파수영역 HRV — Lomb-Scargle PSD (식 13~18)

IBI를 4Hz로 cubic spline 리샘플 → Lomb-Scargle로 PSD (불균등 샘플 OK).

  • • LF = ∫₀.₀₄^₀.₁₅ P(f) df [ms²]
  • • HF = ∫₀.₁₅^₀.₄₀ P(f) df [ms²]
  • • LF/HF 비율 = 논문의 메인 출력 (r=0.84로 가장 강건)

🎯 방향성 일치 — 14명 전원 100%

  • • Baseline → Stress: LF/HF +4.4 (CSI) vs +4.6 (Fitbit)
  • • Stress → Recovery: LF/HF −2.4 (CSI) vs −2.3 (Fitbit)
  • • RMSSD stress 감소율: −47% (CSI) vs −51% (Fitbit)
  • • RMSSD recovery 회복률: +47% (CSI) vs +57% (Fitbit)

→ 절대값보다 추세·방향성이 훨씬 강건. 우리 LF/HF z-score 전략 정확히 정합.

⚠️ Subgroup 결과 — Baseline HR 안정성에 따라 상관 차이
CV<4% (안정) r=0.89 · CV 4~6% (보통) r=0.84 · CV>6% (불안정) r=0.74
→ Beat detection이 불안정 baseline에서 약함. 우리 환자(고령·치매)는 baseline 자체가 불규칙할 수 있어 주의.

⚠️ 논문이 직접 인정한 한계 (§6.3)

  • 1. Reference standard: Smartwatch HRV는 proprietary 알고리즘. ECG gold standard 아님
  • 2. Sample size: N=14는 feasibility pilot 수준, 임상 일반화 불충분
  • 3. Duration: 14분/세션. 일·주 단위 장기 안정성 미검증
  • 4. Task design: 산술 stress는 가벼움. 심한 stress·정서 자극 미평가
  • 5. Environment: 단일 사무실. 5GHz·MIMO·다른 Wi-Fi 인프라 미테스트
  • 6. Population: 건강한 22~46세. 심혈관질환·부정맥 환자 미포함 → 치매·고령 환자에 그대로 적용 불확실

🎯 MajuBom 적용 가이드 — 진짜 베이스 논문

✅ 그대로 이식 가능 (우리 환경과 일치)

  • • 적응형 peak 검출 (μ+1.5σ, 30초 슬라이딩, RRI≥400ms) — Step 1에서 이미 구현 ✓
  • • IQR outlier 제거 (1.5×IQR, 평균 5% 제거) — vital_csi2.py에 없음 → 추가 필요
  • • 위상 차분 + Detrending — vital_csi2.py는 진폭만 씀, 위상 활용 검토
  • • Lomb-Scargle PSD — 불균등 샘플 OK, IBI 시계열에 적합. scipy.signal.lombscargle

⚠️ 우리 환경에서 결정 필요

  • • BPF 차이: 논문 B = 0.8~2.5Hz/4차 vs 논문 A = 1~3.5Hz/2차. 우리는 어느 쪽? → 호흡 잔재 시각화 비교 후 결정
  • • 거리: 논문 B는 3m, 우리는 침대 양쪽 ~1m. 가까울수록 SNR 유리하지만 multipath도 강해짐
  • • 자세: 논문 B는 앉은 자세, 우리는 누운 자세. 가슴 위치 다름 → 호흡 신호 형태 다를 수 있음
  • • 대상: 논문 B는 건강한 청년, 우리는 치매·고령. baseline 불안정 시 r 0.74로 떨어짐 → 캘리브레이션 더 길게

📌 vital_csi3.py 수정 우선순위 (논문 B 기반)

  1. 적응형 peak (μ+1.5σ, 30s window, RRI≥400ms) — Step 1 코드 그대로
  2. IQR outlier 제거 (1.5×IQR) — 새 함수 추가
  3. BPF 0.8~2.5Hz 4차 vs 1~3.5Hz 2차 — A/B 비교 실험

  5. 30일 베이스라인 누적 → 개인별 임계값 자동 조정
