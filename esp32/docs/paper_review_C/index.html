ESP32 CSI + CNN으로 인원수 분류 = MajuBom CSI 파트의 "동행자 존재 여부 확인" (신청서 §주요 기능) 정확히 정합.

🎯 한 줄 요약

ESP32 1개 + 일반 Wi-Fi 라우터로 CSI 수집 → CNN(1D Conv × 2 + Dense × 2)으로 인원수 분류 → 재실유무 98%, 0~2명 96%, 최대 3명 86% 달성.

🔌 시스템 구조 — 기존 연구 vs 이 논문

❌ 기존 연구 한계 (라우터 × 2)

  • • 특정 NIC 칩셋만 작동 (Atheros AR9580/AR9590/AR9344/QCA9558)
  • • 라우터 2개 (TX + RX) 필요
  • • "COTS"라고 하지만 사실상 호환성 제한
  • • 비싸고 셋업 복잡
  • • 상업적 보급 어려움

✅ 이 논문 (ESP32 × 1 + 라우터 × 1)

  • • ESP32 1개 + 집에 있는 일반 Wi-Fi 라우터
  • • 2.4GHz, 802.11n, 20MHz
  • • Sub-carrier 범위 [-28, +28] (총 56개, 52개 사용)
  • • 라우터에 ping → reply에서 CSI 추출
  • • Python 스크립트로 시리얼 데이터 수집
  • • 저렴, 보급 가능
⚠️ 우리(MajuBom) 환경과 다른 점: 이 논문 = ESP32 × 1 + 외부 라우터. 우리 = ESP32 × 2 (TX softAP + RX). → CSI 추출 방식은 유사, 데이터 형식도 호환. CNN 모델 그대로 적용 가능.

📥 데이터 수집 방법

시나리오 5종 (라벨)

  • 👤 0명 (빈 방)
  • 👤 1명
  • 👤👤 2명
  • 👤👤👤 3명
  • 👤👤👤👤 4명

각 시나리오 다양한 행동 (걷기, 앉기, 말하기) 포함 → ML 모델 robust

특징 추출 (CSI raw → feature)

진폭 (Amplitude):
A = √(Re² + Im²)
위상 (Phase):
ϕ = arctan(Im / Re)

진폭 + 위상 둘 다 CNN 입력으로 사용

🧠 CNN 모델 구조

Input
CSI 진폭 + 위상 벡터 (sequential data)
↓
① 1D Conv (64 filters, kernel=10)
+ Batch Normalization + Max Pooling
↓
② 1D Conv (128 filters, kernel=5)
+ Batch Normalization + Max Pooling
↓
③ Flatten
↓
④ Dense (256, ReLU)
+ Batch Norm + Dropout 0.5
↓
⑤ Dense (128, ReLU)
+ Batch Norm + Dropout 0.5
↓
⑥ Softmax (인원수 클래스 출력)

→ 1D Conv 2개로 시계열 특징 추출 → Dense 2개로 분류. 표준적인 CNN-시계열 분류 구조.

⚠️ 논문이 직접 인정한 한계

1.
ESP32의 Sub-carrier 범위 제한

ESP32: [-28, +28] vs 라우터: [-512, +512] → 라우터 대비 정밀도 낮음

2.
단일 안테나 vs 다중 안테나

라우터 다중 안테나로 augment 가능, ESP32는 1안테나라 한계

3.
4명 vs 0명 혼동 (max confusion)

0~4명 분류에서 정확도 급락 (68.22%). 4명 → 0명으로 오분류 빈번.

4.
Overfitting 가능성

단일 환경에서 학습 → 다른 실내 환경에선 재학습 필요

🎯 MajuBom 작업 2에 적용 가이드

✅ 그대로 이식 가능

  • • 1D CNN 모델 구조 그대로 사용 (TensorFlow/PyTorch 둘 다 OK)
  • • 진폭 + 위상 feature 추출 (vital_csi3.py에서 이미 추출 중)
  • • "1명 vs 2명+ 분류"가 우리 목적 → 96.65% 정확도 달성 가능 추정

⚠️ 우리 환경 조정 필요

  • • ESP32 × 2 (softAP) 구성 — 이 논문은 외부 라우터 사용. 우리는 자체 TX softAP. → CSI 수집 코드는 우리 거 그대로 (`csi_recv_router`)
  • • 요양원 환경 재학습 필수 — multipath 지문이 방마다 다름
  • • 침대 누운 자세에서 학습 — 논문은 일반 활동(걷기/앉기) 위주. 우리는 누운 자세 중심

⭐ 핵심 인사이트 — 작업 1과 시너지

작업 2가 "1명 vs 2명+ 감지 (96%)"로 작동하면 → 작업 1 (HRV 추세)의 자동 outlier 게이트 역할:

  • 📍 방에 1명만 → HRV 측정 신뢰 ✓ z-score 적용
  • 📍 방에 2명+ (간병인 등) → HRV 측정 무효화
  • 📍 신청서 §주요 기능(CSI): "심박 변동성(HRV) 측정 + 동행자 존재 여부 확인" 정확히 정합

🗓 구현 우선순위 (발표 후)

  1. 데이터 수집: 1명(누움) × 30회, 2명(누움+옆에 사람) × 30회 = 60개 샘플

    → 각 30초 측정. ESP32 csi_recv_router 그대로 사용

  2. 특징 추출: vital_csi3.py 코드 재사용 (진폭 + 위상)

    → amp = |H|, phase = unwrap(∠H)

  3. CNN 모델 구현: 논문 구조 그대로 (Keras 30줄 정도)

    → 이진 분류 (1명 vs 2명+)이라 더 단순화 가능

  4. 학습 + 검증: 80/20 split, accuracy 측정

    → 목표: 90%+ (논문 96% 기준)

  5. 작업 1과 통합: 게이트 로직 적용

    → "2명+ 감지 시 HRV 측정 무효화"
