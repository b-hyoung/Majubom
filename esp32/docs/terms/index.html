🎯 한 줄로 — 우리가 도대체 뭐 하나

심장이 뛸 때마다 박자 사이 시간이 얼마나 흔들리는지 측정하는 거예요.
그 흔들림이 자율신경 상태를 알려주고, 그게 흐트러지면 낙상 위험 신호일 수 있어요.

💡 비유:

메트로놈이 완벽하게 똑딱똑딱 안 뛰고, 살짝씩 다르게 뛴다고 생각해보세요. 그 "흔들림 폭"이 곧 우리가 보려는 값. WiFi 신호로 가슴 움직임을 감지해 그걸 측정.

🔁 전체 흐름

① 가장 기본

심박 관련 용어

HR (Heart Rate)

단위: bpm

심박수. 1분에 심장이 몇 번 뛰는지.

예: 정상 60~80bpm, 운동 시 120~160bpm

R-R interval / IBI

단위: ms

두 박동 사이 시간 간격. IBI = Inter-Beat Interval.

예: HR 80bpm = R-R 평균 750ms (60000÷80)

NN interval

단위: ms

"정상(Normal) R-R"만. 이상치(부정맥 등)를 뺀 R-R.

실용상 R-R과 거의 같은 의미. 학술 표현.

peak

신호의 산봉우리

CSI 진폭 신호에서 심장이 뛴 순간으로 인식되는 지점. 신호의 국소 최댓값.

peak 시점 차이 = R-R interval

② HRV 본체

HRV 지표 — 두 종류

HRV는 "박동 간격이 얼마나 흔들리는지"를 여러 각도로 본 거예요. 크게 시간영역(시간 단위 통계)과 주파수영역(스펙트럼 분석) 두 가지.

🅰 시간영역 (간단한 통계)

SDNN 단위 ms

R-R 시간들의 표준편차. 전체 변동의 크기.

SDNN = √( Σ(RR_i − RR̅)² / (N−1) )

정상 청년: 30~50ms · 우리 측정: 91.5ms (부풀어 있음)

RMSSD 단위 ms

연속한 두 R-R 차이의 RMS. 빠른 변동만 봄.

RMSSD = √( Σ(RR_{i+1} − RR_i)² / (N−1) )

정상 청년: 20~40ms · 부교감(휴식) 활동 지표

⚠️ 정상은 RMSSD < SDNN. 반대면 peak 잡기 망친 신호

CV 단위 %

변동계수. SDNN을 평균 R-R로 나눈 비율.

CV = SDNN / 평균RR × 100%

🅱 주파수영역 (스펙트럼)

R-R 시퀀스를 푸리에 변환해 어떤 주파수 성분이 강한지 분석

LF 0.04~0.15Hz

저주파 파워. 교감+부교감 혼합 지표.

HF 0.15~0.40Hz

고주파 파워. 부교감(휴식 신경) 지표. 호흡과 관련.

LF/HF 비율

스트레스/자율신경 균형 지표. 흔히 사용.

⭐ 신청서 목표 = 평소 대비 LF/HF z-score 추세

💡 우리 입장:

형석님이 "HRV 절대값(SDNN)에 매달리겠다" 선언 → 🅰 시간영역 메인, 🅱 주파수영역은 백업.

③ 우리 센서

CSI 관련 용어

카메라 대신 WiFi 신호의 미세한 변형으로 가슴 움직임을 추측하는 게 우리 방식. 그 변형 데이터가 CSI.

CSI Channel State Information

WiFi 송신기 → 수신기 사이의 채널 상태 정보. 사람 가슴이 움직이면 채널이 미세하게 바뀜 → 그걸로 심박/호흡 추출.

한 패킷당 64개 subcarrier × (I, Q) 복소수 = 128값

Subcarrier

WiFi(OFDM)는 한 번에 64개의 작은 주파수 채널을 동시에 보냄. 각각이 subcarrier. CSI는 subcarrier별로 따로 나옴.

우리는 그 중 정보 많은 일부만 골라 씀

진폭 (Amplitude)

WiFi 신호의 크기. I, Q값으로 계산: amp = √(I² + Q²)

우리 vital_csi2.py는 진폭만 사용

위상 (Phase)

WiFi 신호 sin파의 시작 위치(각도). 심박 미세진동에 더 민감.

⭐ 논문 B는 위상 사용 (진폭 X). 향후 도입 검토

Quotient model

같은 수신기 두 안테나 CSI를 나눈 것(비율). 공통 노이즈가 약분돼서 깨끗해짐.

⚠️ 우리 ESP32는 안테나 1개라 원본 quotient 불가

Multipath

WiFi 신호가 벽·가구 등에 여러 경로로 반사되어 도착. 환경마다 달라 새 환경에선 정확도 떨어짐.

요양원마다 캘리브레이션 필요한 이유

④ 신호처리

raw → peak 잡기 전 도구들

BPF Band-Pass Filter

특정 주파수 대역만 통과시키는 필터. 라디오 다이얼 같은 거.

우리 vital_csi2.py: 0.9~1.8Hz = 54~108 bpm 대역만 남김 (호흡·노이즈 제거)

논문별 대역 차이: 논문 A=1~3.5Hz, 논문 B=0.8~2.5Hz, 우리=0.9~1.8Hz

Butterworth

가장 흔한 BPF 종류. 대역 안은 평탄, 바깥은 부드럽게 감쇠.

차수가 클수록 더 칼같이 자름 (논문 A는 2차, 논문 B는 4차)

Detrend

신호의 천천히 변하는 추세(드리프트)를 빼서 평평하게 만듦.

하드웨어 온도 변화로 신호가 살살 올라가는 거 제거

PCA Principal Component Analysis

여러 신호를 하나로 압축. 정보가 가장 많이 담긴 방향을 찾아냄.

비유: 합창단 60명 중 멜로디 잘 들리는 1명 자동 선출

우리 vital_csi2.py는 64 subcarrier → PCA로 상위 3개 컴포넌트

z-score / 정규화

데이터를 평균 0, 표준편차 1로 맞춤.

z = (x − μ) / σ

예: 키 175cm → "평균보다 1.2σ 큼"

⚠️ PCA 후 신호는 z-score 분포라 μ+1.5σ 임계가 너무 엄격해짐 (오늘 발견한 문제)

자기상관 autocorrelation

신호가 얼마 만에 자기 자신과 비슷해지나를 찾는 방법.

메트로놈 똑딱 사이 시간을 자동으로 추정. 우리 vital_csi2는 이걸로 평균 심박 주기 T 추정.

⑤ Peak 찾기

신호에서 심장 뛴 시점 잡기

find_peaks

SciPy의 peak 찾기 함수. distance, prominence, height 옵션으로 조건 부과.

scipy.signal.find_peaks

distance

"이 거리 안에 두 peak 있을 수 없음" 제약. 노이즈 peak 줄임.

우리 코드: 자기상관 주기 T × 0.65 (적응형 distance ✅)

prominence

peak가 주변보다 얼마나 솟아있어야 인정할지. 고정값.

⚠️ 고정값이라 신호 크기 변하면 부적합. 우리가 바꾸려는 부분.

μ (뮤)

평균값(mean). 그리스 문자.

σ (시그마)

표준편차(standard deviation). 데이터가 평균에서 얼마나 흩어졌는지.

μ + 1.5σ 적응형 임계

"평균보다 1.5σ 위인 점만 peak로 인정". 신호 크기 따라 임계가 자동 조정.

논문 B 핵심. 우리 PCA z-score 신호엔 너무 엄격함 (실측 결과)

슬라이딩 윈도우 Sliding Window

긴 신호를 일정 크기 창으로 보면서 한 칸씩 옆으로 이동. 각 위치에서 그 구간만의 μ, σ 계산.

예: 90초 신호에 30초 창 → 1초씩 이동하며 60번 통계 계산. 시간 변화 따라잡음.

⑥ 잘못된 값 제거

Outlier 제거

R-R 시퀀스에 가끔 말도 안 되는 값이 들어가요 (peak 놓침/가짜 peak). 예: [750, 780, 760, 2300, 740, 800] ← 2300이 outlier → 통계 망치기 전에 빼야 함.

median 중앙값

데이터 정렬했을 때 한가운데 값. outlier에 강함 (평균은 outlier에 휘둘림).

percentile / quartile

"아래에서 N% 위치 값". 25%(Q1), 50%(중앙값), 75%(Q3).

IQR Interquartile Range

Q3 − Q1. 가운데 50% 데이터의 폭. 데이터 흩어짐을 outlier에 강하게 측정.

IQR = Q3 − Q1

1.5 × IQR 규칙

"Q1 − 1.5×IQR ~ Q3 + 1.5×IQR 밖에 있는 값은 outlier"로 빼버림. 통계 표준.

⭐ 논문 B 식 9의 방법. vital_csi2.py에 추가 필요

median ±30% (우리 현재 방식)

"중앙값에서 ±30% 벗어난 값은 outlier". 정적·임의 임계. IQR보다 단순하지만 분포에 둔감.

vital_csi2.py의 현재 방식. IQR로 교체 검토 중.

⑦ 검증

통계 / 검증 용어

Ground Truth 정답값

"진짜 정답"으로 인정되는 외부 측정값. 우리 CSI 결과를 검증하는 기준.

우리 후보: 애플워치 SDNN (Polar H10이 더 정확하지만 비용)

ECG 심전도

전극으로 심장 전기신호 직접 측정. 가장 정확. HRV의 학술 표준.

병원 검사 / 가슴띠 / 애플워치 ECG 앱(30초)

PPG 광용적맥파

빛으로 혈류 변화 측정 → 심박. 손목 시계·손가락 클립.

ECG보다 부정확. 손목 PPG는 RMSSD에 ±10~30ms 오차

RMSE

오차 제곱평균의 제곱근. 예측-정답 차이의 크기. 큰 오차 더 크게 벌점.

RMSE = √( Σ(pred−true)² / N )

MAE 평균절대오차

예측-정답 차이 절댓값의 평균. 직관적.

MAE = Σ|pred−true| / N

논문 B: SDNN MAE = 5.8ms

r 피어슨 상관계수

-1 ~ 1. 두 값이 같이 움직이는 정도. 1에 가까울수록 일치.

예: r=0.84 → "추세가 매우 잘 따라감"

p-value

"우연일 확률". p < 0.05면 통계적으로 유의(우연 아님).

ICC 급내 상관계수

두 측정 도구가 절대값까지 얼마나 일치하는지. r이 추세만 보는 반면 ICC는 값 자체 일치 평가.

FAQ

자주 헷갈리는 것

Q. SDNN과 RMSSD의 차이가 뭐야?

둘 다 R-R 시간 흔들림을 보는데 보는 시간 스케일이 다름.

SDNN: 모든 R-R 시간이 평균에서 얼마나 흩어져있나 (전체 변동)

RMSSD: 바로 옆 R-R끼리 얼마나 다른가 (빠른 변동만)

비유: 1주일 기온 SDNN = "주 평균 25도, 매일 ±5도 변동" / RMSSD = "어제 27도 오늘 23도, 매일 4도씩 출렁"

Q. 왜 RMSSD > SDNN이면 비정상이야?

RMSSD는 인접 R-R 차이만 보고, SDNN은 전체 흩어짐을 봄.

전체 흩어짐(SDNN)은 장기 변동 + 단기 변동을 다 담는데, RMSSD는 단기 변동만. 보통 단기 ≤ 전체.

실제로 RMSSD > SDNN이 나오면 → peak 잡기에서 박자마다 시간이 마구잡이로 튀고 있다는 신호. 즉 peak 타이밍 잡기가 망가짐.

우리 측정 RMSSD/SDNN = 1.11~1.24 → 약간 비정상. 알고리즘 개선해야 함.

Q. HR과 HRV의 관계?

HR(심박수)은 평균 빠르기. "1분에 80번 뛴다" 같은 평균값.

HRV(심박변이도)는 그 평균에서 박자가 얼마나 흔들리나.

비슷한 HR이라도 HRV는 천차만별. 운동선수 = HR 낮고 HRV 높음(건강), 스트레스 = HR 보통이어도 HRV 낮음.

Q. 왜 적응형 peak가 합성 데이터에선 잘 됐는데 실측에선 망했어?

합성 신호는 raw 진폭이라 큰 값(peak)이 분명히 솟아있음. μ+1.5σ가 그걸 잘 잡음.

실측은 PCA로 z-score 정규화된 신호. 분포가 정규분포(N(0,1))에 가까움.

μ+1.5σ는 정규분포에서 상위 ~7%만 통과 → peak 후보가 너무 적어짐 (11개로 줄어듦).

해결: k값을 1.5 → 0.5나 0.0으로 낮추거나, PCA 전 raw 신호에 적응형 임계 적용.

Q. ECG와 PPG는 뭐가 다르고, 어느 게 더 정확해?

ECG: 심장 전기신호 직접 측정. 정확. 가슴 전극, 의료기기, 가슴띠(Polar H10).

PPG: 빛으로 혈류 변화 측정 → 박동 추정. 손목 시계, 손가락 클립.

R-R 타이밍 정밀도: ECG > PPG. PPG는 ±10~30ms 오차가 일반적.

HRV 학술 연구는 거의 ECG. PPG는 절대값 신뢰는 약하지만 추세 비교엔 쓸 만.

Q. 우리는 결국 뭐 만드는 거야?

침대 양쪽 ESP32-S3 두 대로 WiFi 신호 보내고 받음 → CSI 100Hz 수집

그 신호로 누워있는 사람의 가슴 미세 움직임을 추측 → 심박 R-R 시퀀스 추출

R-R로 SDNN/RMSSD/LF/HF 계산 → 자율신경 상태 추적

평소 대비 변화가 크면 낙상 위험 신호로 간주 → 멀티모달(ToF+mmWave)와 융합해 알람
