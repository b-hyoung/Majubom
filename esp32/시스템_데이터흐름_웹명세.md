# 🌐 MajuBom 시스템 데이터 흐름 + 웹 대시보드 명세

> 작성: 2026-06-15
> 목적: 각 센서(CSI/ToF/mmWave)가 Pi 5 서버에 보낼 **데이터 명세** + 웹 화면 구성
> 박형석(CSI) → 다른 팀원 R&R 공유용 + 통합 작업 가이드

## ⚠️ 운영 환경 — 모든 계산은 Pi 5에서

```
[ESP32]         단순 데이터 수집·송신 (계산 X)
[노트북]        개발/시뮬용 (지금 우리가 쓰는 환경)
[Pi 5]          ⭐ 운영 시 모든 분석/계산/저장/웹 ⭐
```

| 단계 | 지금 (개발) | 운영 (배포 시) |
|---|---|---|
| CSI 수집 | macOS/Win에서 record_csi.py | ESP32 → USB(or WiFi) → Pi 5 |
| 분석 (HR/호흡) | 노트북에서 vital_csi3.py | **Pi 5에서 vital_csi3.py** |
| z-score 계산 | 노트북에서 baseline_zscore.py | **Pi 5** |
| CNN 추론 | 노트북에서 presence_cnn.py | **Pi 5** (학습은 노트북 OK, 모델만 복사) |
| 웹 서버 + 알람 | (없음) | **Pi 5 Flask 풀스택** |
| 데이터 저장 | csv 파일 | **Pi 5 SQLite or 파일** |

→ 노트북 코드 = 그대로 Pi 5에 복사 가능 (Python 동일). 단 PyTorch는 Pi용 설치 필요 (`pip install torch` CPU 버전).

## 🎯 박형석 (CSI) 출력 — 핵심

### 1) HTTP POST 엔드포인트
```
POST http://192.168.0.48:5001/csi
Content-Type: application/json
```

### 2) JSON 데이터 명세

#### Phase 1 — 현재 (raw + 추세)
```json
{
  "timestamp": "2026-06-15T14:30:00Z",
  "bed_id": "bed_01",
  "sensor": "csi",
  "raw": {
    "hr_bpm": 73,                  // 자기상관 HR (위상 기반)
    "resp_rpm": 14,                // 호흡 분당 (Welch PSD)
    "autocorr_strength": 0.45      // 신호 품질 (0~1)
  },
  "quality": {
    "reliable": true,              // strength >= 0.30이면 true
    "samples_count": 9000          // 90초 측정 패킷 수
  }
}
```

#### Phase 2 — 베이스라인 학습 후 (z-score 추가)
```json
{
  "timestamp": "...",
  "bed_id": "bed_01",
  "sensor": "csi",
  "raw": { /* 위와 동일 */ },
  "zscore": {
    "hr": 0.4,                     // 평소 대비 변화량
    "resp": -0.3,
    "strength": 0.2,
    "total_abs": 0.9               // Σ|z|
  },
  "baseline_age_days": 7,          // 베이스라인 누적 일수
  "alert_level": "normal"          // normal / caution / warning / critical
}
```

#### Phase 3 — 작업 2 (재실감지) 학습 후 (인원수 추가)
```json
{
  "timestamp": "...",
  "bed_id": "bed_01",
  "sensor": "csi",
  "raw": { /* ... */ },
  "zscore": { /* ... */ },
  "presence": {
    "count": 1,                    // 1명 or 2명+
    "confidence": 0.96,            // CNN 출력 확률
    "gate_active": true            // 1명일 때만 HRV 추세 신뢰
  },
  "alert_level": "normal"
}
```

### 3) 송신 주기
- **60초마다** 1회 (1분 단위 평균)
- 또는 **이벤트 발생 시** (z-score 임계 초과 등)

### 4) 처리 순서 (CSI 측 펌웨어/스크립트)
```
[ESP32 RX]
  CSI raw 데이터 수집 (~100Hz)
       ↓
  WiFi/MQTT로 Pi에 전달 (csi_http 펌웨어가 처리)
       ↓
[Pi 5 측 — Python 분석기]
  60초 누적 → vital_csi3.py 분석
       ↓
  HR / 호흡 / 강도 추출
       ↓
  베이스라인 비교 → z-score
       ↓
  (Phase 3) CNN 인원 분류
       ↓
  통합 JSON → POST /csi
```

## 🦶 김민혁 (ToF) 출력 — 명세

### 엔드포인트
```
POST http://192.168.0.48:5001/tof
```

### JSON
```json
{
  "timestamp": "2026-06-15T14:30:00.123Z",
  "bed_id": "bed_01",
  "sensor": "tof",
  "sensor_id": "tof1",             // tof1 (머리쪽) / tof2 (발쪽)
  "resolution": "4x4",             // VL53L5CX 모드
  "distances_mm": [234, 456, 123, -1, ..., 16개],  // -1 = 감지 실패 zone
  "targets": [1, 1, 1, 0, ..., 16개],              // 1 = 사람 감지, 0 = 아님
  "state": "lying",                // lying / sitting / standing / leaving / empty
  "trigger": false,                // 침상 이탈 시 true (CSI/mmWave 활성화 신호)
  "fall_detected": false           // 낙상 의심 시 true
}
```

### 송신 주기
- **상시 1~10Hz** (실시간 침상 이탈 감지)
- 상태 변화 시 즉시 추가 송신

### 핵심 출력
- `state`: 환자 자세 (FSM 기반)
- `trigger`: 다른 센서 활성화 신호
- `fall_detected`: 즉시 알람

## 🚶 조우진 (mmWave) 출력 — 명세

### 엔드포인트
```
POST http://192.168.0.48:5001/mmwave
```

### JSON
```json
{
  "timestamp": "...",
  "bed_id": "bed_01",
  "sensor": "mmwave",
  "gait": {
    "speed_mps": 0.65,             // 보행 속도 (정상 0.8~1.2)
    "step_length_cm": 38,          // 보폭
    "step_time_ms": 580,           // 한 걸음 시간
    "lateral_sway_cm": 12.3,       // 좌우 흔들림
    "stv": 0.05,                   // 걸음 변동성 (Step Time Variability)
    "cadence_steps_per_min": 92    // 분당 걸음 수
  },
  "walking_risk_score": 0.35,      // 보행 위험도 (0~1, 0=정상 1=위험)
  "fall_imminent": false,          // 낙상 임박 신호
  "duration_sec": 12               // 보행 측정 시간
}
```

### 송신 주기
- **ToF 트리거 발생 시 활성화** (환자가 침대에서 일어남)
- 보행 구간 종료 시 송신 (예: 5초 이상 보행 감지 후)

### 핵심 출력
- `gait.*`: 보행 패턴 4가지
- `walking_risk_score`: 융합 가중치 70%

## 💻 김도경 (서버/대시보드) — Pi 5에서 통합

### 1) 데이터 수집 (Flask 서버)
```python
# server/main.py
@app.route("/csi", methods=["POST"])
def receive_csi():
    data = request.json
    db.insert("csi", data)
    update_dashboard("csi", data)
    check_alerts(data)
    return "ok"

# /tof, /mmwave 동일 패턴
```

### 2) 통합 위험도 계산
```python
def compute_risk(bed_id):
    csi = db.latest("csi", bed_id)
    tof = db.latest("tof", bed_id)
    mmwave = db.latest("mmwave", bed_id)
    
    # ToF 트리거 없으면 기본 모니터링만
    if not tof["trigger"]:
        return {"level": "normal", "score": 0}
    
    # 융합 (신청서 명세: mmWave 70% + CSI 30%)
    weighted = mmwave["walking_risk_score"] * 0.7 \
             + min(csi["zscore"]["total_abs"] / 4, 1.0) * 0.3
    
    # 4단계 분류
    if weighted < 0.25: level = "normal"
    elif weighted < 0.50: level = "caution"
    elif weighted < 0.75: level = "warning"
    else: level = "critical"
    
    return {"level": level, "score": weighted, "components": {...}}
```

## 🛠 박형석이 다른 팀원에게 제공할 것

### 김도경(서버)에게
```
1. CSI POST 엔드포인트 명세 (위 § "박형석 출력" 참조)
2. zscore.total_abs 계산식 + 임계값
   - normal: < 2
   - caution: 2~4
   - warning: 4~6
   - critical: > 6
3. CSI 측 PyTorch 모델 파일 (presence_cnn.pt)
4. baseline.json 형식 (30일 누적)
```

### 김민혁(ToF)에게
```
1. ToF trigger 신호 = CSI 측정 활성화 조건 합의
   - trigger=true → CSI 측정 시작
   - trigger=false → CSI baseline 누적 모드
2. bed_id 명명 규칙
```

### 조우진(mmWave)에게
```
1. mmWave trigger = ToF trigger 받아 활성화 동일
2. walking_risk_score 계산식 합의 (0~1 정규화)
3. CSI z-score와 cross-check 시점
```
